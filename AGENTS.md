# Todo App Agent Guide

이 파일은 이 저장소에서 Codex가 작업할 때 참고하는 프로젝트 설정 메모입니다.
필요하면 사용자가 직접 수정해서 우선순위와 작업 방식을 바꿀 수 있습니다.

## Product Goal

- 이 프로젝트는 `간단한 todo list 웹 앱`을 만드는 것이 목적이다.
- 기능은 복잡하게 늘리지 말고, 빠르게 쓰기 좋은 개인용 앱을 우선한다.
- 핵심 흐름은 `추가`, `완료`, `삭제`, `필터`, `저장`이다.

## UI Direction

- 화면은 한눈에 이해되도록 단순하게 유지한다.
- 과한 대시보드 스타일보다 입력과 목록이 중심이 되게 만든다.
- 모바일에서도 바로 사용할 수 있어야 한다.

## Coding Preferences

- React + TypeScript 기준으로 작업한다.
- 상태 구조는 가능한 한 단순하게 유지한다.
- 새 기능이 필요하면 먼저 로컬 상태로 해결하고, 외부 라이브러리는 꼭 필요할 때만 추가한다.
- 스타일 수정 시 기존 톤을 크게 깨지 않는 방향을 우선한다.

## Working Rules For Codex

- 새로운 기능을 넣기 전에 기존 todo 흐름을 망가뜨리지 않는지 먼저 본다.
- 사용자가 별도 요청하지 않으면 백엔드나 인증 기능은 추가하지 않는다.
- 설명보다 실제 동작하는 화면과 코드 변경을 우선한다.
- 작업 후 가능하면 `npm run build`로 최소 검증을 한다.
- 모든 작업은 먼저 사용자에게 계획과 영향을 알린다.
- 선택지가 있는 작업은 가능한 옵션을 먼저 제시하고, 사용자가 고를 수 있게 한다.
- 파일 수정, 패키지 설치, 구조 변경, 명령 실행 전에는 무엇을 왜 하는지 먼저 설명한다.
- 사용자가 명시적으로 바로 진행하라고 한 경우를 제외하면, 중요한 변경은 확인 후 진행한다.
- 코드 변경 작업은 수정 전에 변경 계획을 먼저 보여준다.
- 변경 계획에는 수정할 파일, 무엇을 바꿀지, 왜 바꾸는지, 예상 영향 범위를 포함한다.
- 사용자가 확인하기 전에는 코드 수정, 파일 이동, 삭제를 진행하지 않는다.

## User Overrides

아래 항목은 사용자가 자유롭게 바꿔도 된다.

- preferred-language: `ko`
- storage-mode: `localStorage`
- next-priority: `todo ux polish`
- avoid: `heavy state library, login flow, backend setup`
